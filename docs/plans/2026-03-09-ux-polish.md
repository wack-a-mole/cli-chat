# UX Polish — Accessibility & Visual Improvements

**Goal:** Fix text contrast issues, reduce UI noise, and polish the session experience.

**Architecture:** All changes are in the terminal UI layer (src/ui.ts, src/terminal-colors.ts) and the test script. No protocol or server changes needed.

---

### Task 1: Fix text contrast on session backgrounds

**Problem:** `pc.dim()` produces nearly invisible text on dark session backgrounds. Guest side is especially bad — system messages, connection info, and chat text are unreadable.

**Files:**
- Modify: `src/ui.ts`
- Modify: `src/terminal-colors.ts`
- Test: `src/__tests__/ui.test.ts`

**Approach:**
- Add a `sessionLog()` helper that uses `pc.white()` (bright white) instead of `pc.dim()` for all session output when a background is active
- Replace `pc.dim()` calls in `showSystem()`, `showToolUse()`, `showToolResult()`, `showTurnComplete()` with contrast-aware versions
- For `showWelcome()`: use bright white for labels, bright green for the join command, bright yellow for slack message
- Keep `pc.dim()` as fallback when no background is active (normal terminal)
- Update `showUserPrompt()` colors: use `pc.whiteBright` + bold for labels, white for text
- The `showApprovalRequest()` yellow text is okay — keep it

**Key code changes in `src/ui.ts`:**
- Add method: `private fmt(text: string, color: (s: string) => string, dimFallback?: boolean): string` — applies bright color when background active, dim when not
- `showSystem()`: use `pc.white` instead of `pc.dim` when background active
- `showUserPrompt()`: use `pc.whiteBright` + `pc.bold` for user label, `pc.white` for message text
- All `pc.dim()` calls → use `pc.gray` as minimum (still readable on dark bg)
- Slack message line: `pc.white` not `pc.dim`

### Task 2: Clear screen on session start

**Problem:** Shell noise (Last login, You have mail, node version, zsh prompt) clutters the terminal before session UI appears.

**Files:**
- Modify: `src/ui.ts`

**Approach:**
- In `applySessionBackground()`, after setting the background, write `\x1b[2J\x1b[H` (clear screen + move cursor to top)
- This gives the session a clean, dedicated feel
- Both host and guest get a clean slate

**Code:**
```typescript
applySessionBackground(): void {
  if (this.background) return;
  this.background = pickSessionBackground();
  // Clear screen and move cursor to top, then apply background
  process.stdout.write("\x1b[2J\x1b[H");
  process.stdout.write(applyBackground(this.background));
}
```

### Task 3: Silent auto-approval (reduce noise)

**Problem:** Every guest message on host side shows:
```
⚠ Benji wants to send: "message"
  (Approval handling via TUI — auto-approving for now)
```
This doubles every guest message visually. Since we auto-approve, just show the message directly.

**Files:**
- Modify: `src/ui.ts`

**Approach:**
- `showApprovalRequest()`: remove the two noisy log lines, just silently call the approval handler
- The message itself is already displayed by `showUserPrompt()` in `host.ts` line 88
- Result: guest messages appear once (as a user message), not twice

**Code:**
```typescript
showApprovalRequest(promptId: string, user: string, text: string): void {
  // Auto-approve silently — the message is already shown via showUserPrompt
  if (this.approvalHandler) {
    this.approvalHandler(promptId, true);
  }
}
```

### Task 4: Add input prompt indicator

**Problem:** Just a bare blinking cursor — users don't know where to type.

**Files:**
- Modify: `src/ui.ts`

**Approach:**
- After the readline `line` event fires and message is processed, write a subtle prompt
- Use `⟩ ` as the prompt character (matches Claude Code style)
- Show it after welcome, after each message display

**Code:**
```typescript
private showPrompt(): void {
  if (this.background) {
    process.stdout.write(pc.gray("⟩ "));
  }
}
```
Call `showPrompt()` at end of `startInputLoop` (initial), and after processing each line.
